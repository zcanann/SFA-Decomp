## In this codebase

The wiki page describes the shipped `/streams/*.adp` assets from the outside (disc layout).
This repo has the *code* that opens and plays them, plus several adjacent tables that let us
verify (and in a few cases correct) the wiki's directory-name guesses.

### The stream-playback system

- **`StreamEntry`** — `include/main/engine_shared.h:327-334` — the 0x16-byte record loaded from
  the runtime stream table (id, fadeBits, volBits, lengthRaw, `name[0xF]`, flag). `name` is what
  gets concatenated into the DVD path, so it's expected to carry the directory prefix, e.g.
  `"ARW/2cloud"`, `"CC/caplight"` (15 bytes is enough for every filename in the wiki's tables).
  ```c
  typedef struct {
      u16 id;          // 0x0  - looked up by AudioStream_Play(id, ...)
      u8 fadeBits;      // 0x2  - >>6&3 / >>4&3 index gAudioStreamFadeTable {0,2,4}; >>2&3 = stop-sfx
      u8 volBits;       // 0x3  - bit7 selects channel mask; bits0-6 = base volume (0-127)
      u16 lengthRaw;    // 0x4  - stream length, divided by a frames/sec constant for gAudioStreamEndPos
      char name[0xF];   // 0x6  - disc-relative name, e.g. "ARW/2cloud" (no directory separator assumed elsewhere)
      u8 flag;          // 0x15 - zeroed for every entry once the table finishes loading
  } StreamEntry;
  ```
- **`AudioStream_Play(int id, void (*preparedCallback)(void))`** — `src/main/audio.c:2167` — the
  central entry point. Walks `gStreamsData[0..gStreamsCount)` for a matching `id`, then builds
  the disc path with `concatThreeStrings(path, 0x40, "/streams/", s->name, ".adp")` and calls
  `DVDOpen`/`DVDPrepareStreamAsync`. This is the literal code that turns a `StreamEntry` into one
  of the wiki's `<DIR>/<name>.adp` files.
- **The `/streams/` prefix is verified in our own decompiled data**, not assumed: MWCC's string-pool
  merging left it as a trailing substring of an unrelated warning string —
  `src/main/audio.c:3283`: `char sDvdCancelStreamWarning[0x3C] = "WARNING:DVDCancelStreamAsync returned FALSE\n\0\0\0\0/streams/";`
  (symbol `sDvdCancelStreamWarning` at `.data:0x802C5DC4`, size `0x3C`, per
  `config/GSAE01/symbols.txt:10255`). `AudioStream_Play` reads the `/streams/` tail via
  `(char*)fadeTbl + 0x3C` where `fadeTbl == gAudioStreamFadeTable` (`.data:0x802C5DB8`), i.e.
  `gAudioStreamFadeTable + 0x3C == sDvdCancelStreamWarning + 0x30`.
- **`.adp` extension**: `extern char sAdpExtension;` used at `src/main/audio.c:2214`; the actual
  symbol is `sAdpExtension = .sdata:0x803DB254; size:0x5 data:string` (`config/GSAE01/symbols.txt:12713`),
  i.e. `".adp\0"` — matches the wiki's "standard GameCube ADP files" line exactly.
- **The stream *table itself*** (not the `.adp` audio, the index of id/name/size/flags) is loaded
  from a sibling data file, also recovered from a merged string literal —
  `src/main/audio.c:3052-3053`:
  ```c
  char sMusicTriggersLoadedCallbackLoadError[356] =
      "musicTriggersLoadedCallback load error\n\0/audio/data/Music.bin\0\0\0"
      "/audio/data/Sfx.bin\0/audio/data/Streams.bin\0"
      "audioInit: sndIsInstalled() returned FALSE!\n\0\0\0\0"
  ```

  `audioLoadTriggerData()` (`src/main/audio.c:2864-2886`) loads `base+0x1e0` (`/audio/data/Streams.bin`)
  into `gStreamsData`/`gStreamsCount` via `streamsLoadedCallback` (`src/main/audio.c:2034`), and the
  sibling `base+0x1b4`/`base+0x1cc` paths (`/audio/data/Music.bin`, `/audio/data/Sfx.bin`) are the
  *music-trigger* and *sfx-trigger* tables — a separate, looping-music system (see below), not ADP
  streaming. `Streams.bin` is the on-disc table this wiki page's directory listing was presumably
  built from (or reverse-engineered against); we don't have its contents extracted in this repo, only
  the record layout (`StreamEntry`) and the code that walks it.
- **Globals**: `gStreamsData`/`gStreamsCount` (`include/main/engine_shared.h:862`), plus the whole
  `gAudioStream*` family (`gAudioStreamCurrentId`, `gAudioStreamPreparedId`, `gAudioStreamDvdState`,
  `gAudioStreamPos`/`gAudioStreamEndPos`, `gAudioStreamVolumeLeft/Right`, `gAudioStreamMusicFadeFlagA/B`,

  `AudioStream_StopAll`, `AudioStream_StartPrepared`, `AudioStream_CancelPrepared`,
  `AudioStream_PrepareCallback`, `AudioStream_PlayAddrCallback`, `AudioStream_IsPreparing`,
  `AudioStream_GetCurrentId`, `AudioStream_SetVolume`/`SetDefaultVolume`,
  `AudioStream_GetMusicFadeFlagA`/`B`, `AudioStream_UpdateFadeTimer`, `AudioStream_Init`.
- **Low-level DVD/AI streaming** (separate from the `.adp` id lookup above, but what it rides on
  top of): `DVDPrepareStreamAsync`/`DVDPrepareStreamAbsAsync`/`DVDCancelStreamAsync`/
  `DVDStopStreamAtEndAsync`/`DVDGetStreamPlayAddrAsync`/`DVDLowAudioStream` and
  `AISetStreamPlayState`/`AIGetStreamPlayState`/`AISetStreamSampleRate`/`AISetStreamVolLeft`/
  `AISetStreamVolRight` (all in `config/GSAE01/symbols.txt`, Dolphin SDK). Distinct again from
  `src/main/audio/hw_stream.c`'s `hwFlushStream`/`hwGetPos`/`hwInitStream`/`hwExitStream` /
  `src/main/audio/aram_data.c`'s `aramGetStreamBufferAddress` — that's the ARAM ring-buffer path
  for *synth-engine* (MusyX) streamed samples, not the DVD `/streams/*.adp` cutscene system.

### Not the same system: `Music_Trigger` / `MUSICTRIG_*`

`include/main/audio/music_trigger_ids.h` defines `MUSICTRIG_*` IDs (index into `/audio/data/Music.bin`,
named after the looping `.sng` background-music track each one selects) passed to `Music_Trigger()`.
These are a **separate looping-music system**, not the one-shot `.adp` dialogue streams this wiki page
covers — but the two are frequently triggered together. E.g. `AudioStream_Play` special-cases stream
id `1318` (`src/main/audio.c:2184-2188`, matching the wiki's `KP/kp1318.adp`) to also fire
`Music_Trigger(MUSICTRIG_drako_3, 0)` and `Music_Trigger(MUSICTRIG_TTH_Night, 1)` — a KP-directory
voice line cross-triggering a ThornTail-Hollow-named music cue, presumably because that cutscene plays
over a scene transition between the two areas. `MUSICTRIG_KP_Text`, `MUSICTRIG_TTH_Fight`,
`MUSICTRIG_TTH_Night`, `MUSICTRIG_DIM_Cavern`/`DIM_Snow`, `MUSICTRIG_CRF_Swim`/`Suspense`,
`MUSICTRIG_WLC_*` (Walled City), `MUSICTRIG_LVF_Tracking` and others in that header directly reuse
the same area abbreviations as this wiki's stream directories (KP, TTH, DIM, CRF, WLC/WC, LV/LVF).

### Callers (what actually plays a stream, and which wiki directory it lines up with)

- **`src/main/dll/ARW/dll_02A1_arwlevelcon.c`** (`arwlevelcon_update`) — one instance per on-rails
  Arwing course; `ARWLevelConState.streamId` (set per `mapEventSlot`) is passed straight to
  `AudioStream_Play`. This is the code behind the wiki's five `ARW/2*.adp` course files.
- **`src/main/objseq.c`** — the cutscene/map-event sequencer. `gObjSeqStreamTableA`/`gObjSeqStreamTableB`
  (per-track stream-id tables, looked up via `seqStreamLookupFn_8007fff8`) drive `AudioStream_Play`
  at `objseq.c:1417` and `objseq.c:4048`, keyed by `trackId` derived from the current sequence slot.
  This is the general "area cutscene voice line" path that most of the wiki's per-level directories
  (CC, CRF, DIM, DR, LV, MMP, TTH, WC, ...) are driven through.
- **`src/main/dll/maybetemplate.c`** (`gameTextFn_80125ba4`) — the pause-menu "comms" head-display
  popup. `gHeadDisplayEntryTable` is a table of `HEADREC_STRIDE`(0xC)-byte records
  (`HEADREC_STREAM_ID`@0, `HEADREC_TEXT_ID`@4, `HEADREC_PANEL_TYPE`@6, `HEADREC_NPC_DIALOGUE`@7,
  `HEADREC_BOX`@8); entry `idx`'s stream id is passed to `AudioStream_Play`.
- **`src/main/dll/dll_0000_gameui.c`** — pause-menu Communicator hint lines: literal ids in the
  `0x2710`-`0x273a` range (e.g. `0x2714`, `0x271a`, `randomGetRange(0,1)+0x271b`, `+0x2727`, `+0x2739`)
  — this is the code behind the wiki's `COMM/*_pe.adp`/`*_sl.adp`/`*_gp.adp`/`*_rb.adp` hint lines
  (the `_pe`/`_sl`/`_gp`/`_rb` filename suffixes are per-character voice variants).
  `pauseMenuFn_8012b77c` picks between three of these ranges by `pauseMenuState`.
- **`src/main/dll/player.c:13315`** and **`src/main/dll/dll_010E_deathseq.c:94`** — one-off ids
  `0x51e0`/`0x51e1` played on player death/respawn — not covered by any per-area directory in the
  wiki's listing (these ids are well outside every numeric `.adp` filename in the tables above), so
  they likely live under a directory the wiki page doesn't enumerate, or an id not tied to a
  literal filename number at all (recall `StreamEntry.name` need not match `StreamEntry.id`).

SP, VF, WC, WM) is a **related but separately-assigned** area-code scheme — it overlaps with several
wiki stream-directory names (ARW, DIM, DR, MMP, VF≈VFP, WC) but not all: `dll/SH` groups SnowHorn-area
scripted objects (`shtricky`, `shswapston`) while the wiki's `SSH` (SwapStone Hollow) and `SW` (SnowHorn

the wiki's `DFS` (a Krazoa shrine). Don't assume a `dll/XX` folder and a wiki `XX` stream directory name
the same place.

## Ready-to-adopt code

The wiki page itself is a pure asset listing (filenames/sizes/guessed directory meanings) — it doesn't
define an enum, flag table, or id list that maps cleanly onto a single header. The one piece of *our*
code this cross-reference makes concrete enough to name is `StreamEntry.fadeBits`/`.volBits`, whose bit
layout is currently only implicit in `AudioStream_Play`'s shift/mask expressions
(`include/main/engine_shared.h` / `src/main/audio.c:2252-2258`):

```c
/* StreamEntry.fadeBits bit layout (src/main/audio.c AudioStream_Play) */
#define STREAM_FADEBITS_FLAGA_SHIFT 6   /* >>6 & 3, indexes gAudioStreamFadeTable {0,2,4} -> gAudioStreamMusicFadeFlagA */
#define STREAM_FADEBITS_FLAGB_SHIFT 4   /* >>4 & 3, indexes gAudioStreamFadeTable {0,2,4} -> gAudioStreamMusicFadeFlagB */
#define STREAM_FADEBITS_STOPSFX_SHIFT 2 /* >>2 & 3, non-zero stops all object sfx (Sfx_StopAllObjectSounds) */

/* StreamEntry.volBits bit layout */
#define STREAM_VOLBITS_CHANMASK_BIT 7   /* >>7 & 1: 1 -> gAudioActiveChannelMask = 4, else 0 */
#define STREAM_VOLBITS_VOLUME_MASK 0x7F /* base volume 0-127, scaled by gAudioStreamDefaultVolume >> 7 */
```

This is offered for a maintainer to fold into `include/main/engine_shared.h` alongside `StreamEntry`
if/when that improves readability there — not applied here per this doc's scope.
